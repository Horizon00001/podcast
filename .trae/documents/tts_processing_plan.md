# 播客脚本 TTS 自动合成计划

我们将编写一个 Python 脚本，读取 `output/podcast_script.json`，并使用 `edge-tts` 库将文本转换为语音。

## 1. 环境准备
- 使用清华源安装 `edge-tts` 库。
- 确保系统已安装 Python 3.x。

## 2. 核心功能实现 (`tts_synthesize.py`)
- **角色配音映射**：
  - `主持人A` (小凯) -> `zh-CN-YunxiNeural` (男声，热情)
  - `主持人B` (小琳) -> `zh-CN-XiaoxiaoNeural` (女声，亲切/活泼)
- **脚本解析**：
  - 遍历 JSON 中的 `sections` 和 `dialogues`。
  - 提取 `speaker` 和 `content`。
- **语音合成**：
  - 异步调用 `edge-tts` 生成音频流。
  - 暂时将所有语音合并为一个完整的 MP3 文件，或者按对话生成独立文件（根据复杂度决定，优先生成单一完整音频）。
- **音效处理**：
  - 在生成过程中，如果遇到 `audio_effect`，在终端输出提示信息（目前 `edge-tts` 不支持自动混入背景音乐，需后续手动混音或使用其他工具）。

## 3. 输出目录
- 所有生成的音频文件将存放在 `output/audio/` 目录下。

## 4. 验证步骤
- 运行脚本。
- 检查 `output/audio/` 目录下是否生成了对应的 MP3 文件。
- 播放音频确认声音是否与角色匹配，内容是否完整。

## 5. 假设与决策
- 假设用户系统中已配置好 Python 环境。
- 决策：为了保持简单高效，初版脚本将按顺序合成所有对话并拼接成一个音频文件（如果库支持流式拼接），或者为每个对话生成带编号的文件，方便后续混音。
