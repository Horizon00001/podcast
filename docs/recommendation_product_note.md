# 推荐系统产品说明草案

## 目标

- 提升推荐准确度，让推荐更接近用户真实兴趣。
- 减少只看点击带来的噪声，降低误触对结果的干扰。
- 让推荐结果具备可解释性，用户能理解“为什么推荐给我”。

## 现在会看什么

- 播放、点赞、收藏、跳过等行为。
- 内容的播放时长、是否听完、是否重复收听。
- 跳过发生在什么位置。
- 用户最近一段时间连续在听什么。
- 用户是在什么时间段收听。

## 为什么要加这些信号

- 只看“点没点”很容易把误触、随手打开、浅尝辄止算成兴趣。
- 收听时长、完播率、重复收听能更准确地反映真实偏好。
- 跳过位置能区分“刚打开就不感兴趣”和“听到一半后离开”。
- 连续行为序列能反映近期兴趣变化。
- 时间上下文能捕捉用户在早上、通勤、晚上等不同场景下的偏好差异。

## 数据使用原则

- 只采集和推荐直接相关的数据。
- 不采集无关的手机个人信息。
- 不把所有点击都当成强兴趣。
- 对跳过、点赞、收藏等行为做区分，避免误判。

## 推荐结果会怎么变化

- 听得更久、听完、反复收听的内容，会更容易被推荐。
- 很早跳过的内容，会更快被降权。
- 最近连续收听的主题，会比很久以前的行为更重要。
- 不同时间段的偏好会影响推荐排序，让结果更贴近当下场景。

## 产品侧建议

- 增加“个性化推荐”开关，允许用户关闭。
- 增加“重置推荐”能力，方便用户清空历史偏好。
- 在推荐卡片上展示“推荐理由”，增强透明度。
- 冷启动阶段同时保留“热门推荐”和“个性化推荐”。

## 一句话总结

- 推荐不再只看用户点了什么，而是更看用户怎么听、听了多久、在哪儿停、什么时候听，这样推荐会更准、更稳定。
